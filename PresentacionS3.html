Paquetes esenciales: tidyverse y dplyr

El ecosistema tidyverse

Tidyverse es una colección de paquetes de R diseñados para ciencia de datos que comparten una filosofía de diseño, gramática y estructuras de datos.

Ventajas de tidyverse
  • Consistencia: Sintaxis y estructura unificada
  • Operador pipe (%>%): Secuencias de operaciones intuitivas
  • Legibilidad: Código más fácil de entender y mantener
  • Eficiencia: Operaciones optimizadas para grandes conjuntos de datos
  • Comunidad activa: Excelente documentación y soporte

dplyr: El paquete esencial para manipulación de datos

dplyr proporciona una "gramática" para la manipulación de datos con verbos intuitivos que corresponden a las operaciones más comunes:

Carga de paquetes y operaciones básicas con dplyr
# Cargar tidyverse (incluye dplyr) library(tidyverse) # Verbos principales de dplyr con el dataset de pingüinos # 1. filter(): Filtrar filas según condiciones pingüinos_adelie <- penguins %>% filter(species == "Adelie") # 2. select(): Seleccionar columnas medidas_pingüinos <- penguins %>% select(species, island, bill_length_mm, flipper_length_mm, body_mass_g) # 3. mutate(): Crear o modificar columnas penguins_con_indice <- penguins %>% mutate( bill_body_ratio = bill_length_mm / body_mass_g, body_mass_kg = body_mass_g / 1000 ) # 4. arrange(): Ordenar filas penguins_ordenados <- penguins %>% arrange(desc(body_mass_g)) # 5. summarise(): Resumir datos resumen_por_especie <- penguins %>% group_by(species) %>% summarise( peso_promedio = mean(body_mass_g, na.rm = TRUE), max_longitud_aleta = max(flipper_length_mm, na.rm = TRUE), n_pingüinos = n() )
Consejo: El operador pipe (%>%) permite encadenar operaciones de manera intuitiva. Se lee como "y luego". Simplifica el código al evitar variables intermedias y anidamiento de funciones.

Variables cualitativas y cuantitativas

Variables cualitativas (categóricas)

Expresan características o cualidades que no pueden medirse con números.

  • Nominales: No tienen orden intrínseco (ej. especies, islas)
  • Ordinales: Tienen un orden natural (ej. rangos, niveles)

En el dataset de pingüinos:

  • species: Nominal (Adelie, Chinstrap, Gentoo)
  • island: Nominal (Biscoe, Dream, Torgersen)
  • sex: Nominal (male, female)
Variables cuantitativas (numéricas)

Expresan cantidades medibles numéricamente.

  • Discretas: Valores enteros, contables
  • Continuas: Cualquier valor en un intervalo

En el dataset de pingüinos:

  • bill_length_mm: Continua
  • bill_depth_mm: Continua
  • flipper_length_mm: Continua
  • body_mass_g: Continua

Escalas de medición

Las cuatro escalas de medición

Las escalas de medición definen el nivel de información que contienen las variables y las operaciones matemáticas permitidas.

Escala Características Operaciones Ejemplos en pingüinos Representación en R
Nominal Categorías sin orden Igualdad/desigualdad (=, ≠) species, island, sex factor (sin ordered)
Ordinal Categorías con orden Mayor/menor (>, <) Tamaño (si lo clasificáramos como pequeño, mediano, grande) ordered factor
Intervalo Distancias iguales, cero arbitrario Suma/resta (+, -) year (año de observación) numeric
Razón Distancias iguales, cero absoluto Todas (incl. multiplicación/división) bill_length_mm, body_mass_g numeric

Diferencias entre escalas y su importancia

Jerarquía de escalas de medición
¿Por qué importa la escala?
  • Determina qué operaciones matemáticas son válidas
  • Define qué estadísticos son significativos (no tiene sentido calcular la media de una variable nominal)
  • Influye en la elección de técnicas analíticas y gráficas
  • Afecta la interpretación de los resultados

Tablas de frecuencia en R

Tipos de tablas de frecuencia

Las tablas de frecuencia son herramientas fundamentales para resumir variables categóricas, mostrando cuántas veces ocurre cada valor o categoría.

Conceptos clave
  • Frecuencia absoluta: Conteo de cada categoría
  • Frecuencia relativa: Proporción respecto al total (0-1)
  • Frecuencia porcentual: Porcentaje del total (0-100%)
  • Frecuencia acumulada: Suma progresiva de frecuencias
Distribución de especies

Creación de tablas de frecuencia en R

Con funciones base de R
# Tabla de frecuencia absoluta freq_especies <- table(penguins$species) print(freq_especies) # Tabla de frecuencia relativa freq_rel_especies <- prop.table(freq_especies) print(freq_rel_especies) # Tabla de frecuencia porcentual freq_porc_especies <- prop.table(freq_especies) * 100 print(freq_porc_especies) # Tabla de frecuencia acumulada freq_acum_especies <- cumsum(freq_especies) print(freq_acum_especies) # Tabla de frecuencia completa tabla_completa <- data.frame( Frecuencia = freq_especies, Porcentaje = round(freq_porc_especies, 2), Acumulada = freq_acum_especies, Porc_Acum = round(cumsum(freq_porc_especies), 2) ) print(tabla_completa)
Con tidyverse (dplyr)
# Tabla de frecuencia con dplyr tabla_especies <- penguins %>% count(species) %>% mutate( porcentaje = round(n / sum(n) * 100, 2), acumulado = cumsum(n), porc_acum = round(cumsum(porcentaje), 2) ) print(tabla_especies) # Tabla de frecuencia para isla tabla_islas <- penguins %>% count(island) %>% mutate( porcentaje = round(n / sum(n) * 100, 2), acumulado = cumsum(n), porc_acum = round(cumsum(porcentaje), 2) ) print(tabla_islas)

Tablas de contingencia en R

¿Qué son las tablas de contingencia?

Las tablas de contingencia (o tablas cruzadas) muestran la relación entre dos o más variables categóricas, permitiendo identificar patrones y asociaciones entre ellas.

Estructura de una tabla de contingencia

Creación de tablas de contingencia en R

Con funciones base de R
# Tabla de contingencia básica: Especie vs Isla tabla_cont_especies_islas <- table(penguins$species, penguins$island) print(tabla_cont_especies_islas) # Añadir marginales (totales por fila y columna) tabla_cont_marginales <- addmargins(tabla_cont_especies_islas) print(tabla_cont_marginales) # Proporciones por fila (% dentro de cada especie) prop_filas <- prop.table(tabla_cont_especies_islas, 1) * 100 print(round(prop_filas, 2)) # Proporciones por columna (% dentro de cada isla) prop_columnas <- prop.table(tabla_cont_especies_islas, 2) * 100 print(round(prop_columnas, 2)) # Proporciones totales (% del total general) prop_totales <- prop.table(tabla_cont_especies_islas) * 100 print(round(prop_totales, 2))
Con tidyverse (dplyr)
# Tabla de contingencia con dplyr tabla_especies_islas <- penguins %>% count(species, island) %>% pivot_wider( names_from = island, values_from = n, values_fill = 0 ) print(tabla_especies_islas) # Tabla completa con porcentajes por fila tabla_porcentajes <- penguins %>% count(species, island) %>% group_by(species) %>% mutate( total_especie = sum(n), porcentaje = round(n / total_especie * 100, 2) ) %>% ungroup() %>% arrange(species, island) print(tabla_porcentajes)

Análisis de asociación

Prueba de independencia Chi-cuadrado
# Prueba Chi-cuadrado para evaluar asociación prueba_chi <- chisq.test(tabla_cont_especies_islas) print(prueba_chi) # Interpretación de resultados # Si p-valor < 0.05: Existe asociación estadísticamente significativa # Si p-valor ≥ 0.05: No hay evidencia suficiente de asociación

Gráficos circulares

Los gráficos circulares (o de pastel) muestran las proporciones de las categorías como porciones de un círculo completo.

Creación de gráficos circulares con ggplot2
# Preparar datos para el gráfico circular datos_pie <- penguins %>% count(species) %>% mutate( porcentaje = round(n / sum(n) * 100, 1), etiqueta = paste0(species, "\n", porcentaje, "%") ) # Gráfico circular con ggplot2 p5 <- ggplot(datos_pie, aes(x = "", y = n, fill = species)) + geom_bar(width = 1, stat = "identity") + coord_polar("y", start = 0) + labs( title = "Proporción de especies de pingüinos", fill = "Especie" ) + theme_minimal() + theme( axis.title = element_blank(), axis.text = element_blank(), panel.grid = element_blank() ) + geom_text( aes(label = etiqueta), position = position_stack(vjust = 0.5) ) + scale_fill_manual(values = c("#6200ea", "#00e5ff", "#00c853")) # Mostrar gráfico p5
Proporción de especies de pingüinos
Advertencia sobre gráficos circulares: Los gráficos circulares pueden ser visualmente atractivos, pero tienen limitaciones para la interpretación precisa de datos. El cerebro humano percibe mejor las diferencias en longitud (como en los gráficos de barras) que en ángulos o áreas (como en los circulares). Use gráficos circulares principalmente cuando:
  • Tenga pocas categorías (≤ 5)
  • Quiera enfatizar proporciones respecto al total
  • Las diferencias entre categorías sean sustanciales
Para análisis más precisos, se recomiendan los gráficos de barras.

# Crear tablas de contingencia para relaciones interesantes tabla_esp_isla <- table(penguins_limpio$species, penguins_limpio$island) tabla_esp_sexo <- table(penguins_limpio$species, penguins_limpio$sex) tabla_esp_tamaño <- table(penguins_limpio$species, penguins_limpio$size_category) # Mostrar tablas con marginales addmargins(tabla_esp_isla) addmargins(tabla_esp_sexo) addmargins(tabla_esp_tamaño) # Pruebas de independencia Chi-cuadrado prueba_esp_isla <- chisq.test(tabla_esp_isla) prueba_esp_sexo <- chisq.test(tabla_esp_sexo) prueba_esp_tamaño <- chisq.test(tabla_esp_tamaño) # Mostrar resultados de las pruebas print(prueba_esp_isla) print(prueba_esp_sexo) print(prueba_esp_tamaño)
