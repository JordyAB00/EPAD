Importación y exportación de datos

Proceso de importación de datos

El primer paso en cualquier análisis estadístico es la obtención y carga de los datos. En nuestro caso, importaremos el dataset del Titanic.

Pasos para la importación

Preparar el archivo CSV

El archivo "titanic.csv" contiene datos sobre los pasajeros del Titanic. Asegúrate de que el archivo esté en tu directorio de trabajo o especifica la ruta completa.

Instalar y cargar paquetes necesarios

Utilizaremos el paquete tidyverse (que incluye readr, dplyr, ggplot2) para importar y manipular nuestros datos.

# Instalación y carga de paquetes install.packages("tidyverse") library(tidyverse)
Importar el archivo CSV

Utilizando la función read_csv() de readr, importaremos nuestro dataset. Esta función es parte del ecosistema tidyverse y proporciona mejor rendimiento que read.csv() de R base.

Examinar la estructura de los datos

Una vez importados los datos, examinaremos su estructura con funciones como glimpse(), str(), head() y summary().

Vista previa del dataset
Métodos de importación en R
  • read_csv(): Para archivos CSV (readr)
  • read_excel(): Para archivos Excel (readxl)
  • read_tsv(): Para archivos TSV (readr)
  • read.csv(): Método base R para CSV
  • read.table(): Método general base R

Exportación de datos

Tras realizar algunas transformaciones, podemos exportar nuestros datos para uso futuro:

  • CSV: write_csv(data, "nombre_archivo.csv")
  • Excel: write_xlsx(data, "nombre_archivo.xlsx") (requiere el paquete writexl)
  • RDS: saveRDS(data, "nombre_archivo.rds") (formato nativo de R)
Consejo práctico: El formato RDS de R conserva todos los tipos de datos y atributos específicos de R, lo que lo hace ideal para guardar datos procesados entre sesiones.

Paquetes esenciales: tidyverse y dplyr

El ecosistema tidyverse

Tidyverse es una colección de paquetes de R diseñados para ciencia de datos con una filosofía común de diseño.

  • dplyr: Manipulación de datos
  • ggplot2: Visualización de datos
  • tidyr: Reorganización de datos
  • readr: Importación de datos
  • purrr: Programación funcional
  • tibble: Marcos de datos modernos
Ventajas de tidyverse:
  • Sintaxis consistente y declarativa
  • Uso del operador pipe (%>%) para encadenar operaciones
  • Funciones optimizadas para tareas comunes
  • Excelente integración entre paquetes

dplyr: Manipulación de datos

dplyr ofrece un conjunto de "verbos" que facilitan las tareas más comunes de manipulación de datos:

Función Descripción Equivalente SQL
filter() Filtrar filas según condiciones WHERE
select() Seleccionar columnas SELECT
mutate() Crear o modificar columnas Expresiones en SELECT
arrange() Ordenar filas ORDER BY
summarise() Resumir datos Funciones de agregación
group_by() Agrupar datos para operaciones GROUP BY
join() Combinar datasets (left_join, inner_join, etc.) Varios tipos de JOIN

El operador pipe (%>%)

El operador pipe (%>%) permite encadenar operaciones de manera intuitiva, mejorando la legibilidad del código:

# Enfoque tradicional (anidado) resultado <- summarise( group_by( filter(titanic_data, Age > 18), Pclass ), promedio_tarifa = mean(Fare, na.rm = TRUE) ) # Con operador pipe resultado <- titanic_data %>% filter(Age > 18) %>% group_by(Pclass) %>% summarise(promedio_tarifa = mean(Fare, na.rm = TRUE))
Flujo de trabajo recomendado con tidyverse:
  1. Importar datos con read_csv() o similar
  2. Transformar datos con verbos de dplyr
  3. Visualizar resultados con ggplot2
  4. Comunicar hallazgos con R Markdown

Variables cualitativas y cuantitativas

Clasificación de variables

La clasificación correcta de las variables es fundamental para determinar los métodos estadísticos adecuados para su análisis.

Variables cualitativas (categóricas)

Expresan cualidades o categorías, no cantidades medibles numéricamente.

  • Nominales: Sin orden natural (sexo, puerto de embarque, cabina)
  • Ordinales: Con orden natural (clase de pasajero: 1ª, 2ª, 3ª)

En nuestro dataset:

  • Survived, Sex, Embarked (nominales)
  • Pclass (ordinal)
Variables cuantitativas (numéricas)

Expresan cantidades medibles numéricamente.

  • Discretas: Valores enteros, contables (número de hermanos/cónyuges)
  • Continuas: Cualquier valor en un intervalo (edad, tarifa)

En nuestro dataset:

  • Age, Fare (continuas)
  • SibSp, Parch (discretas)

Identificando el tipo de variable en R

Verificación de la clase de las variables

R asigna diferentes clases a las variables según su tipo:

  • character: Cadenas de texto
  • factor: Variables categóricas
  • numeric: Números (enteros o decimales)
  • integer: Números enteros específicamente
  • Date: Fechas
Conversión entre tipos

A menudo necesitamos convertir variables entre diferentes tipos:

# Convertir texto a factor titanic_data$Sex <- as.factor(titanic_data$Sex) # Convertir numeric a factor con orden titanic_data$Pclass <- factor(titanic_data$Pclass, levels = c(1, 2, 3), ordered = TRUE) # Crear categorías a partir de variables numéricas titanic_data$Age_Group <- cut(titanic_data$Age, breaks = c(0, 12, 18, 35, 60, Inf), labels = c("Niño", "Adolescente", "Joven adulto", "Adulto", "Mayor"))
Importancia para el análisis: El tipo de variable determina qué estadísticos calcular y qué gráficos utilizar. Para variables categóricas utilizaremos frecuencias, moda y gráficos de barras, mientras que para variables numéricas usaremos media, mediana, desviación estándar, histogramas, etc.

Escalas de medición

Las cuatro escalas fundamentales

Las escalas de medición definen el nivel de información contenido en una variable y determinan qué operaciones matemáticas y estadísticas son válidas.

Escala Características Operaciones permitidas Ejemplos en nuestro dataset
Nominal Categorías sin orden Igualdad/desigualdad (=, ≠) Sex, Embarked, Survived
Ordinal Categorías con orden Mayor/menor (>, <) Pclass (1ª, 2ª, 3ª clase)
Intervalo Distancias iguales, cero arbitrario Suma/resta (+, -) (No hay claros ejemplos en el dataset)
Razón Distancias iguales, cero absoluto Todas (incl. multiplicación/división) Age, Fare, SibSp, Parch
Jerarquía de escalas de medición
Cada nivel superior incluye todas las propiedades de los niveles inferiores
Estadísticos apropiados por escala
  • Nominal: Moda, frecuencias, proporciones
  • Ordinal: Mediana, percentiles, rango
  • Intervalo: Media, desviación estándar
  • Razón: Media geométrica, coeficiente de variación

Aplicación en el dataset del Titanic

En nuestro análisis:

  • Para variables nominales como Sex y Embarked: analizaremos frecuencias y representaremos con gráficos de barras o circulares
  • Para variables ordinales como Pclass: estudiaremos la distribución por categorías y examinaremos su relación con otras variables
  • Para variables de razón como Age y Fare: podemos calcular medias, desviaciones estándar, y representar con histogramas o boxplots
Error común: Calcular medias para variables nominales u ordinales. Por ejemplo, si codificamos Survived como 0=No, 1=Sí, la "media" puede interpretarse como proporción de supervivientes, pero para Pclass (1,2,3), la media no tiene una interpretación directa significativa.

Tablas de frecuencia en R

¿Qué son las tablas de frecuencia?

Las tablas de frecuencia son herramientas fundamentales para resumir y presentar datos categóricos, mostrando cuántas veces ocurre cada valor o categoría en el conjunto de datos.

Tipos de frecuencias
  • Frecuencia absoluta: Conteo de casos en cada categoría
  • Frecuencia relativa: Proporción respecto al total (0-1)
  • Frecuencia porcentual: Porcentaje del total (0-100%)
  • Frecuencia acumulada: Suma progresiva de frecuencias
Ejemplo de tabla de frecuencia

Creación de tablas de frecuencia en R

Con funciones base de R

R base ofrece funciones simples pero efectivas para crear tablas de frecuencia:

# Tabla de frecuencia absoluta freq <- table(titanic_data$Pclass) # Tabla de frecuencia relativa prop.table(freq) # Tabla de frecuencia porcentual prop.table(freq) * 100 # Frecuencia acumulada cumsum(freq)
Con dplyr (tidyverse)

dplyr ofrece un enfoque más versátil y con mejor formato:

# Tabla de frecuencia completa con dplyr titanic_data %>% count(Pclass) %>% mutate( porcentaje = round(n / sum(n) * 100, 2), acumulado = cumsum(n), porc_acum = round(cumsum(porcentaje), 2) )

Aplicaciones en nuestro dataset

En nuestro análisis, crearemos tablas de frecuencia para variables categóricas clave:

  • Distribución de supervivencia (Survived: 0=No, 1=Sí)
  • Frecuencia por clase de pasajero (Pclass: 1, 2, 3)
  • Distribución por sexo (Sex: male, female)
  • Frecuencia por puerto de embarque (Embarked: C, Q, S)

Tablas de contingencia en R

¿Qué son las tablas de contingencia?

Las tablas de contingencia (o tablas cruzadas) muestran la relación entre dos o más variables categóricas, permitiéndonos examinar cómo se distribuyen conjuntamente sus valores.

Creación de tablas de contingencia en R

Con funciones base de R
# Tabla de contingencia básica (sexo vs supervivencia) tabla_cont <- table(titanic_data$Sex, titanic_data$Survived) # Añadir totales marginales addmargins(tabla_cont) # Proporciones por fila (% dentro de cada sexo) prop.table(tabla_cont, 1) * 100 # Proporciones por columna (% dentro de cada grupo de supervivencia) prop.table(tabla_cont, 2) * 100
Con dplyr y tidyr
# Tabla de contingencia con dplyr titanic_data %>% count(Sex, Survived) %>% pivot_wider( names_from = Survived, values_from = n, values_fill = 0 )

Análisis de asociación

Las tablas de contingencia nos permiten examinar si existe asociación entre variables categóricas:

# Prueba Chi-cuadrado para evaluar asociación chisq.test(tabla_cont) # Interpretación de resultados # Si p-valor < 0.05: Existe asociación estadísticamente significativa # Si p-valor ≥ 0.05: No hay evidencia suficiente de asociación

Visualización de variables categóricas con ggplot2

Importancia de la visualización

La visualización efectiva de datos categóricos facilita la identificación de patrones, proporciones y relaciones que pueden no ser evidentes en tablas numéricas.

Gráficos de barras

Los gráficos de barras son la forma más común y efectiva de visualizar variables categóricas, mostrando frecuencias o proporciones.

Gráfico de barras básico
# Gráfico de barras básico ggplot(titanic_data, aes(x = Pclass, fill = factor(Pclass))) + geom_bar() + labs(title = "Distribución por clase de pasajero", x = "Clase", y = "Frecuencia") + theme_minimal()

Gráficos de barras agrupadas

Para visualizar la relación entre dos variables categóricas:

Gráfico de barras agrupadas
# Gráfico de barras agrupadas ggplot(titanic_data, aes(x = Pclass, fill = factor(Survived))) + geom_bar(position = "dodge") + labs(title = "Supervivencia por clase", x = "Clase", y = "Frecuencia") + theme_minimal()

Gráficos circulares

Los gráficos circulares (o de pastel) muestran las proporciones como segmentos de un círculo.

Gráfico circular
# Preparar datos para gráfico circular class_counts <- titanic_data %>% count(Pclass) %>% mutate(porcentaje = n / sum(n) * 100) # Crear gráfico circular ggplot(class_counts, aes(x = "", y = n, fill = factor(Pclass))) + geom_bar(stat = "identity", width = 1) + coord_polar("y", start = 0) + labs(title = "Distribución por clase de pasajero") + theme_minimal()
Limitaciones de los gráficos circulares: Aunque son visualmente atractivos, los gráficos circulares tienen limitaciones para la interpretación precisa. El cerebro humano percibe mejor las diferencias en longitud (gráficos de barras) que en ángulos o áreas (gráficos circulares). Use gráficos circulares principalmente cuando:
  • Tenga pocas categorías (≤ 5)
  • Quiera enfatizar proporciones respecto al total
  • Las diferencias entre categorías sean sustanciales
