El operador pipe (%>%)

El operador pipe (%>%) permite encadenar operaciones de manera intuitiva, mejorando la legibilidad del código:

# Enfoque tradicional (anidado) resultado <- summarise( group_by( filter(datos, temperatura > 25), ciudad ), temp_promedio = mean(temperatura) ) # Con operador pipe resultado <- datos %>% filter(temperatura > 25) %>% group_by(ciudad) %>% summarise(temp_promedio = mean(temperatura))

Variables cualitativas y cuantitativas

Clasificación de variables

La clasificación correcta de las variables es fundamental para determinar los métodos estadísticos adecuados para su análisis.

Variables cualitativas (categóricas)

Expresan cualidades o categorías, no cantidades medibles numéricamente.

  • Nominales: Sin orden natural (ciudad, país, condición meteorológica)
  • Ordinales: Con orden natural (intensidad del viento: baja, media, alta)

En nuestro dataset:

  • city, country (nominales)
  • weather_condition (nominal)
Variables cuantitativas (numéricas)

Expresan cantidades medibles numéricamente.

  • Discretas: Valores enteros, contables (número de días de lluvia)
  • Continuas: Cualquier valor en un intervalo (temperatura, humedad)

En nuestro dataset:

  • temp_max, temp_min (continuas)
  • humidity, wind_speed (continuas)

Identificando el tipo de variable en R

Verificación de la clase de las variables

R asigna diferentes clases a las variables según su tipo:

  • character: Cadenas de texto
  • factor: Variables categóricas
  • numeric: Números (enteros o decimales)
  • integer: Números enteros específicamente
  • Date: Fechas
Conversión entre tipos

A menudo necesitamos convertir variables entre diferentes tipos:

# Convertir texto a factor datos$weather_condition <- as.factor(datos$weather_condition) # Convertir texto a fecha datos$date <- as.Date(datos$date) # Crear categorías a partir de variables numéricas datos$temp_category <- cut(datos$temp_max, breaks = c(-Inf, 10, 20, 30, Inf), labels = c("Frío", "Templado", "Cálido", "Muy caliente"))
Importancia para el análisis: El tipo de variable determina qué estadísticos calcular y qué gráficos utilizar. Para variables categóricas utilizaremos frecuencias, moda y gráficos de barras, mientras que para variables numéricas usaremos media, mediana, desviación estándar, histogramas, etc.

Escalas de medición

Las cuatro escalas fundamentales

Las escalas de medición definen el nivel de información contenido en una variable y determinan qué operaciones matemáticas y estadísticas son válidas.

Escala Características Operaciones permitidas Ejemplos en nuestro dataset
Nominal Categorías sin orden Igualdad/desigualdad (=, ≠) city, country, weather_condition
Ordinal Categorías con orden Mayor/menor (>, <) Intensidad del viento (si la clasificáramos)
Intervalo Distancias iguales, cero arbitrario Suma/resta (+, -) temperatura en °C (el cero es arbitrario)
Razón Distancias iguales, cero absoluto Todas (incl. multiplicación/división) humedad (%), velocidad del viento (km/h)
Jerarquía de escalas de medición
Cada nivel superior incluye todas las propiedades de los niveles inferiores
Estadísticos apropiados por escala
  • Nominal: Moda, frecuencias, proporciones
  • Ordinal: Mediana, percentiles, rango
  • Intervalo: Media, desviación estándar
  • Razón: Media geométrica, coeficiente de variación

Aplicación en el dataset meteorológico

En nuestro análisis:

  • Para variables nominales como weather_condition: analizaremos frecuencias y representaremos con gráficos de barras o circulares
  • Para variables de razón como humidity: podemos calcular medias, desviaciones estándar, y representar con histogramas o boxplots
  • Para temperaturas (variables de intervalo): analizaremos medias, rangos y distribuciones

Tablas de frecuencia en R

¿Qué son las tablas de frecuencia?

Las tablas de frecuencia son herramientas fundamentales para resumir y presentar datos categóricos, mostrando cuántas veces ocurre cada valor o categoría en el conjunto de datos.

Tipos de frecuencias
  • Frecuencia absoluta: Conteo de casos en cada categoría
  • Frecuencia relativa: Proporción respecto al total (0-1)
  • Frecuencia porcentual: Porcentaje del total (0-100%)
  • Frecuencia acumulada: Suma progresiva de frecuencias
Ejemplo de tabla de frecuencia

Creación de tablas de frecuencia en R

Con funciones base de R

R base ofrece funciones simples pero efectivas para crear tablas de frecuencia:

# Tabla de frecuencia absoluta freq <- table(weather_data$weather_condition) # Tabla de frecuencia relativa prop.table(freq) # Tabla de frecuencia porcentual prop.table(freq) * 100 # Frecuencia acumulada cumsum(freq)
Con dplyr (tidyverse)

dplyr ofrece un enfoque más versátil y con mejor formato:

# Tabla de frecuencia completa con dplyr weather_data %>% count(weather_condition) %>% mutate( porcentaje = round(n / sum(n) * 100, 2), acumulado = cumsum(n), porc_acum = round(cumsum(porcentaje), 2) )

Aplicaciones en nuestro dataset

En nuestro análisis, crearemos tablas de frecuencia para variables categóricas clave:

  • Frecuencia de diferentes condiciones meteorológicas (Soleado, Nublado, Lluvioso, etc.)
  • Distribución de observaciones por país
  • Frecuencia de rangos de temperatura (agrupando temperaturas en categorías)

Tablas de contingencia en R

¿Qué son las tablas de contingencia?

Las tablas de contingencia (o tablas cruzadas) muestran la relación entre dos o más variables categóricas, permitiéndonos examinar cómo se distribuyen conjuntamente sus valores.

Creación de tablas de contingencia en R

Con funciones base de R
# Tabla de contingencia básica (país vs condición meteorológica) tabla_cont <- table(weather_data$country, weather_data$weather_condition) # Añadir totales marginales addmargins(tabla_cont) # Proporciones por fila (% dentro de cada país) prop.table(tabla_cont, 1) * 100 # Proporciones por columna (% dentro de cada condición) prop.table(tabla_cont, 2) * 100
Con dplyr y tidyr
# Tabla de contingencia con dplyr weather_data %>% count(country, weather_condition) %>% pivot_wider( names_from = weather_condition, values_from = n, values_fill = 0 )

Análisis de asociación

Las tablas de contingencia nos permiten examinar si existe asociación entre variables categóricas:

# Prueba Chi-cuadrado para evaluar asociación chisq.test(tabla_cont) # Interpretación de resultados # Si p-valor < 0.05: Existe asociación estadísticamente significativa # Si p-valor ≥ 0.05: No hay evidencia suficiente de asociación

Visualización de variables categóricas con ggplot2

Importancia de la visualización

La visualización efectiva de datos categóricos facilita la identificación de patrones, proporciones y relaciones que pueden no ser evidentes en tablas numéricas.

Gráficos de barras

Los gráficos de barras son la forma más común y efectiva de visualizar variables categóricas, mostrando frecuencias o proporciones.

Gráfico de barras básico
# Gráfico de barras básico ggplot(weather_data, aes(x = weather_condition, fill = weather_condition)) + geom_bar() + labs(title = "Distribución de condiciones meteorológicas", x = "Condición", y = "Frecuencia") + theme_minimal()

Gráficos de barras agrupadas

Para visualizar la relación entre dos variables categóricas:

Gráfico de barras agrupadas
# Gráfico de barras agrupadas ggplot(weather_data, aes(x = continent, fill = weather_condition)) + geom_bar(position = "dodge") + labs(title = "Condiciones por continente", x = "Continente", y = "Frecuencia") + theme_minimal()

Gráficos circulares

Los gráficos circulares (o de pastel) muestran las proporciones como segmentos de un círculo.

Gráfico circular
# Preparar datos para gráfico circular weather_counts <- weather_data %>% count(weather_condition) %>% mutate(porcentaje = n / sum(n) * 100) # Crear gráfico circular ggplot(weather_counts, aes(x = "", y = n, fill = weather_condition)) + geom_bar(stat = "identity", width = 1) + coord_polar("y", start = 0) + labs(title = "Distribución de condiciones meteorológicas") + theme_minimal()
Limitaciones de los gráficos circulares: Aunque son visualmente atractivos, los gráficos circulares tienen limitaciones para la interpretación precisa. El cerebro humano percibe mejor las diferencias en longitud (gráficos de barras) que en ángulos o áreas (gráficos circulares). Use gráficos circulares principalmente cuando:
  • Tenga pocas categorías (≤ 5)
  • Quiera enfatizar proporciones respecto al total
  • Las diferencias entre categorías sean sustanciales
