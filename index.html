¿Qué es la estadística?

La estadística es la ciencia que se ocupa de la recolección, organización, análisis, interpretación y presentación de datos.

Estadística descriptiva

Se centra en resumir, organizar y describir un conjunto de datos mediante medidas numéricas y visuales.

Estadística inferencial

Utiliza métodos para hacer deducciones, predicciones y generalizaciones sobre una población basándose en una muestra.

La estadística proporciona las herramientas fundamentales que sustentan gran parte de la metodología utilizada en la ciencia de datos moderna.

Analítica descriptiva

"La analítica descriptiva responde a la pregunta: ¿Qué ha ocurrido?"

  • Se centra en resumir los datos históricos
  • Proporciona contexto y entendimiento básico
  • Utiliza medidas estadísticas, tablas y visualizaciones
  • Es el fundamento para análisis más avanzados

Técnicas y herramientas

  • Medidas de tendencia central y dispersión
  • Tablas de frecuencia y tablas cruzadas
  • Visualizaciones: histogramas, gráficos de barras, diagramas de caja
  • Dashboards y reportes automatizados

Analítica predictiva

"La analítica predictiva responde a la pregunta: ¿Qué es probable que ocurra?"

  • Utiliza datos históricos para predecir eventos futuros
  • Se basa en modelos estadísticos y algoritmos de ML
  • Identifica patrones y relaciones en los datos
  • Cuantifica la probabilidad de resultados futuros

Técnicas y herramientas

  • Regresión lineal y logística
  • Series temporales (ARIMA, Prophet)
  • Árboles de decisión y bosques aleatorios
  • Redes neuronales y deep learning
Semana 1: Panorama general de la analítica de datos

Analítica prescriptiva

"La analítica prescriptiva responde a la pregunta: ¿Qué debemos hacer?"

  • No solo predice lo que ocurrirá, sino que recomienda acciones
  • Evalúa diferentes escenarios y sus resultados
  • Considera restricciones y objetivos de negocio
  • Representa el nivel más avanzado de analítica

Técnicas y herramientas

  • Optimización y programación lineal
  • Simulación de Monte Carlo
  • Sistemas de recomendación
  • Inteligencia artificial avanzada

El ciclo de análisis de datos es iterativo, con frecuentes retornos a fases anteriores para refinamiento.

Semana 1: Ciclo de vida de un proyecto de análisis de datos

Fase 1: Definición del problema

"Todo análisis efectivo comienza con una pregunta clara."

  • Identificar el problema de negocio a resolver
  • Definir objetivos específicos y medibles
  • Establecer el alcance del análisis
  • Identificar a los stakeholders clave
  • Determinar los recursos necesarios

Ejemplo

Una cadena minorista quiere entender los factores que afectan la retención de clientes para diseñar mejores programas de fidelización.

Herramientas y técnicas

  • Entrevistas con stakeholders
  • Análisis de requisitos
  • Marcos SMART para objetivos
  • Análisis de viabilidad
  • Definición de KPIs
Semana 1: Ciclo de vida de un proyecto de análisis de datos

Fase 2: Recolección de datos

"La calidad de tu análisis nunca será mejor que la calidad de tus datos."

  • Identificar fuentes de datos relevantes
  • Evaluar la disponibilidad y accesibilidad de los datos
  • Considerar aspectos éticos y legales (privacidad, consentimiento)
  • Establecer procedimientos de recolección
  • Documentar la procedencia y características de los datos

Ejemplo

Para analizar la retención, la empresa recopila datos de su sistema CRM, historial de compras, interacciones con servicio al cliente y encuestas de satisfacción.

Fuentes de datos

  • Internas (bases de datos, sistemas transaccionales, archivos)
  • Externas (API, datasets públicos, proveedores)
  • Web scraping
  • Sensores e IoT
  • Encuestas y focus groups
Semana 1: Ciclo de vida de un proyecto de análisis de datos

Fase 3: Limpieza y preparación de datos

"La limpieza de datos suele representar el 80% del tiempo en un proyecto de análisis."

  • Identificar y manejar valores faltantes
  • Detectar y corregir errores y inconsistencias
  • Identificar y tratar valores atípicos (outliers)
  • Transformar variables según sea necesario
  • Integrar datos de múltiples fuentes
  • Crear variables derivadas relevantes

Ejemplo

Se unifican los datos del cliente, se corrigen errores en los identificadores, se eliminan duplicados y se crean variables como "tiempo desde última compra" y "valor total del cliente".

Herramientas comunes

  • R (dplyr, tidyr, data.table)
  • Python (pandas, numpy)
  • SQL para transformaciones
  • OpenRefine
  • Herramientas ETL especializadas
Semana 1: Ciclo de vida de un proyecto de análisis de datos

Fase 4: Exploración y análisis de datos

"El análisis exploratorio revela patrones y relaciones que guían análisis más profundos."

  • Calcular estadísticas descriptivas básicas
  • Explorar distribuciones de variables
  • Examinar relaciones entre variables
  • Identificar patrones, tendencias y anomalías
  • Formular y probar hipótesis
  • Aplicar técnicas analíticas apropiadas según el problema

Ejemplo

El análisis revela que los clientes tienden a abandonar después de 3 meses sin compras, y que la retención es mayor en clientes que han interactuado con el servicio al cliente.

Técnicas analíticas

  • Análisis univariado y bivariado
  • Análisis de correlación
  • Segmentación y clustering
  • Análisis de series temporales
  • Modelado predictivo
  • Minería de texto
Semana 1: Ciclo de vida de un proyecto de análisis de datos

Fase 5: Visualización de datos

"Una visualización efectiva revela la historia detrás de los datos de un vistazo."

  • Seleccionar el tipo de visualización apropiado para los datos
  • Crear gráficos claros y efectivos
  • Diseñar dashboards interactivos
  • Asegurar que las visualizaciones sean accesibles
  • Evitar distorsiones y representaciones engañosas

Ejemplo

Se crea un dashboard interactivo mostrando la tasa de retención por segmento de cliente, con la capacidad de filtrar por región, canal de compra y otros factores relevantes.

Tipos de visualizaciones

  • Gráficos de barras y líneas
  • Histogramas y diagramas de caja
  • Gráficos de dispersión
  • Mapas de calor
  • Gráficos de área y de radar
  • Mapas geoespaciales
Semana 1: Ciclo de vida de un proyecto de análisis de datos

Fase 6: Comunicación e implementación

"Los mejores análisis fracasan si no pueden comunicarse e implementarse efectivamente."

  • Traducir hallazgos técnicos a lenguaje de negocios
  • Desarrollar narrativas convincentes basadas en datos
  • Presentar resultados de manera clara y efectiva
  • Proporcionar recomendaciones accionables
  • Implementar hallazgos en procesos y sistemas
  • Medir el impacto de las acciones tomadas

Ejemplo

Se presenta un informe ejecutivo con recomendaciones específicas: implementar un programa de recompensas para clientes inactivos por 2+ meses y mejorar el seguimiento post-venta.

Componentes clave

  • Resumen ejecutivo
  • Metodología utilizada
  • Hallazgos principales
  • Visualizaciones impactantes
  • Recomendaciones específicas
  • Plan de implementación
  • Métricas de seguimiento
